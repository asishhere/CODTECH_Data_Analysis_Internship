# Task 3: Dashboard Development — Website Traffic Sources

## 📝 Project Overview
An interactive web interface that tracks multi-channel digital acquisition performance across real-time operations[cite: 1]. The application utilizes interactive UI callbacks to manipulate complex data sets, mapping traffic attributes directly into interactive dashboards[cite: 1].

## 💻 System Design
*   **Layout Component Layer:** Configured purely in Python via Dash DOM nodes wrapped inside responsive grids[cite: 1].
*   **Reactive Callback Layer:** Real-time state listening that catches input changes (dropdown updates) and triggers targeted backend computation to rebuild chart states[cite: 1].

## 🛠️ Technical Stack
*   **Framework:** Plotly Dash, Flask[cite: 1]
*   **UI Components:** Dash Bootstrap Components (Dark Theme Framework)[cite: 1]
*   **Engine:** Pandas, NumPy[cite: 1]

*   **KPI Tracking:** Dynamic metrics reporting total sessions, absolute conversions, mean bounce rates, and session durations[cite: 1].
*   **Temporal Sessions Trend:** Multi-line timeline showing acquisition shifts[cite: 1].
*   **Traffic Allocation Share:** Donut charts breaking down marketing channel weight[cite: 1].
